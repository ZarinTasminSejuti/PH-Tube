Q.1: Discuss the scope of var, let, and const

- var scope: var keyword is usually used for denoting variables. var are mainly used as global scope or function scope. In function scope means, they can access within the entire function and also fro the outside of the function block. var also has hoisting criteria.

- let scope: let keyword is usually used for denoting variables in modern Javascript (ES6). let are mainly used as block scope that means it can not be accessed outside from the function block.

- const scope: const keyword is usually used for denoting variables in modern Javascript (ES6). const are mainly used as block scope. Like let variable, const also can not be accessed outside from the function block.

Q.2: Tell us the use cases of null and undefined

Undefined:

  1. If a variable is declared but not initialized then the value occurs as undefined.
  2. function with no return gets undefined
  3. parameter that is not passed will be undefined
  4. when property doesn't exists on an object will give undefined value.
  5. set a value directly as undefined will get undefined

Null: Empty or absence of object value is null value.

Q.3: What do you mean by REST API?

REST API is a HTTP methods architecture. There are four HTTP methods describes in below,

- GET Method: This methods are receive information of API resource.

- POST Method: This methods are create an API resource.

- PUT/PATCH Method: This methods are update an API resource.

- DELETE Method: This methods are delete an API resource.
